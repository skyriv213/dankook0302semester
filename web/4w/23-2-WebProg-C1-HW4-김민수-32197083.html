문맥교환이란?

하나의 프로세스가 CPU를 사용 중인 상태에서 다른 프로세스가 CPU를 사용하도록 하기 위해,
이전의 프로세스의 상태(문맥)을 보관하고 새로운 프로세스의 상태를 적재하는 작업을 말함

발생 시점

    멀티테스킹
  • 실행 가능한 프로세스들이 운영체제의 스케줄러에 의해 번갈아가며 수행
  • 번갈아 가면서 프로세스가 CPU를 할당 받는데 이때 context switching이 발생한다

    • 인터럽트 핸들링
  • 예외 상황 발생 시 CPU에게 해당 사항을 알리고 처리하는 것
  • 해당 인터럽트가 발생 시 문맥교환이 실행

    • 사용자와 커널 모드 전환
  • 사용자와 커널 모드 전환은 문맥교환이 필수가 아니지만 운영체제에 따라 발생할 수 있다


예시 및 사진

  1. P1이 실행되는 도중 인터럽트 or 시스템 콜 발생

  2. ProcessControlBlock(PCB)에 해당 P1의 정보를 저장

  3. 실행하고자 하는 P2의 상태를 불러옴

  4. P2의 실행

  5. P2가 실행이 되다가 다시 인터럽트 or 시스템콜 발생

  6. PCB에 다시 P2의 정보를 저장

  7. P1을 다시 해당 PCB에서 불러와서 실행


문맥교환 장단점

문맥교환의 장단점
장점 단점
여러 개의 프로세스를 동시에 실행하여
멀티태스킹을 가능하게 함		 문맥교환은 기본적으로 오버헤드가 발생하여
성능 저하의 문제가 발생할 수 있음
문맥교환의 시간이 길어지면 시스템 응답 시간이
늦어질 수 있음
